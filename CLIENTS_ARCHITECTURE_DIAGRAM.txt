QUERY FLOW EXAMPLE:
═══════════════════════════════════════════════════════════════════

1. User visits /clients
2. Frontend calls: GET /api/clients?page=1&limit=10
3. API validates request & auth token
4. API executes SQL:
   - Filters deleted_at IS NULL
   - Filters company_id = user's company
   - Orders by created_at DESC
   - LIMIT 10 OFFSET 0
5. Returns: { items: [], total: N, page: 1, limit: 10, pages: N }
6. Frontend renders table with results
7. User can search, filter, create, update, or delete
8. All changes go through same flow

STATUS CODES:
═════════════════════════════════════════════════════════════════

✅ 200 OK         - Successful GET, PUT, DELETE
✅ 201 Created    - Successful POST
⚠️ 400 Bad Req    - Validation error (bad email, missing field)
⚠️ 401 Unauthorized - No/invalid token
⚠️ 403 Forbidden  - Token valid but no permission
⚠️ 404 Not Found  - Client doesn't exist or is deleted
❌ 500 Error      - Server-side error (rare with proper validation)


SECURITY LAYERS:
═══════════════════════════════════════════════════════════════════

Layer 1: Authentication
         └─ Check JWT token valid
         
Layer 2: Authorization
         └─ Check user owns the client (user_id match)
         └─ Check client is in user's company (company_id match)
         
Layer 3: Row Level Security (RLS)
         └─ Database-level access control
         └─ Cannot bypass with SQL injection
         
Layer 4: Input Validation
         └─ Email format validation
         └─ Phone format validation
         └─ Budget range validation
         └─ Required field checks
         
Layer 5: Rate Limiting
         └─ Prevent brute force attacks
         └─ Limit API calls per user
